¿Cómo trabajamos?

Definición de una idea

Nace la creación de una idea a partir de una necesidad de negocio

Especificaciones del proyecto

Se especifican los alcances del proyecto, tecnologías de desarrollo, metodologías, requerimientos, requisitos, limitantes, etc. siempre teniendo como objetivo cubrir las necesidades de nuestros clientes

Priorizar backlog

Se realiza una lista de los requerimientos a cubrir y se ordenan a partir de los más prioritarios; esto con el objetivo de enfocarnos a lo que es más importante para el cliente, y entregar así primero lo que genera más valor para el negocio.

Diseño de interfaces

Se establecen reuniones periódicas con el negocio para mostrar distintos prototipos de diseño, a partir de las especificaciones definidas

Implementación

Se desarrollan los módulos con base a lo planeado (de acuerdo al orden de prioridades) y se van entregando resultados de acuerdo a las iteraciones de entrega definidas, así el cliente puede visualizar avances del proyecto, identificar y prever posibles riesgos.

Revisión del entregable

Se asegura que cada unidad de desarrollo así como la integración de todos los componentes trabajen en conjunto de una forma integral y continua, de acuerdo a las expectativas del cliente. Adicionalmente, se implementan pruebas adicionales tales como las pruebas de calidad, de estrés, de carga, etc. para garantizar la calidad del producto.

Pruebas y calidad

Se valida que en cada entrega se cumplan los estándares de calidad esperados por el cliente. Adicionalmente, se revisa que la entrega cumpla con los requerimientos del plan de trabajo

Desarrollo de software

El sistema se implementa en un ambiente para revisión del cliente.

Revisión del producto

El cliente da el visto bueno (vo.bo.) del entregable y da la aprobación para pasarlo a un ambiente productivo.

Periodo de garantía

Se establece un tiempo de garantía a partir de la fecha de implementación en un ambiente productivo para asegurar el éxito de la entrega

Producto liberado

El producto está implementado en un ambiente productivo y funciona de acuerdo a lo planeado. Si el cliente identifica mejoras y/o requiere cualquier tipo de asistencia, puede contactarnos a través de nuestro sitio web Devstudio Helpdesk

Revisión de alcance

Se revisa que los cambios solicitados estén alineados de acuerdo al plan de trabajo. Cualquier cambio de impacto se analiza para incrementar el alcance del producto
